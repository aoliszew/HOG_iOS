# Heart of Gold — iOS driving companion

A Hitchhiker's-Guide-themed spaceship bridge app: every drive is a space mission.
Feature vision: `HOG_iOS_Feature_Requests.rtf`. Phased plan: `ROADMAP.md`.
Current status/task board: `docs/TASKS.md`.

> This file is the single source of truth for ALL coding agents — Claude Code
> reads it via CLAUDE.md, Codex reads it natively. Change workflow rules here,
> in one place only.

## Build & run

- Project is generated by **XcodeGen**: after adding/removing files, run `xcodegen generate`.
  Never hand-edit `HeartOfGold.xcodeproj` — edit `project.yml` instead.
- Build: `xcodebuild -project HeartOfGold.xcodeproj -scheme HeartOfGold -destination 'platform=iOS Simulator,name=iPhone 16' build`
- Speed simulation for testing: Simulator → Features → Location → Freeway Drive.
- No third-party dependencies yet. Do not add packages without noting it in docs/TASKS.md.

## Event content

Encounters/quests are data, not code: JSON in `Content/events/` following
`docs/EVENT_SYSTEM.md`. Validate with `python3 tools/validate_events.py` before
any content PR (CI enforces it). Content must work audio-only for the driver.

## Architecture (respect these seams)

- `ShipController` is the single coordinator; views talk only to it.
- `EventSource` protocol (Events/EventEngine.swift) is the seam for story generation.
  Phase 3's Claude-backed generator implements it; don't bypass it.
- `VoiceSynthesizing` protocol (Voice/ShipVoice.swift) is the seam for TTS upgrades.
- Audio session config lives only in `Audio/AudioEngine.swift` — one owner, category
  `.playback` with `.duckOthers` so we work behind Apple Maps. Don't create other sessions.
- Safety rule: the driver's UI must never require reading/typing while moving; all
  driver-facing features are audio-first.

## Multi-agent workflow

These rules apply to every agent (Claude, Codex, or otherwise) and to humans:

- `main` is protected by convention: never commit directly; always work on a branch
  named `<agent>/<topic>` (e.g. `claude/voice-commands`, `codex/app-icon`) and open a PR.
- Before starting, claim a task in `docs/TASKS.md`: put your branch name in the
  "Claimed by" column and push that as the first commit on your branch. Check the
  board (and open PRs) first — if a task is claimed, pick another. Unclaim by PR
  if you abandon a task.
- Keep PRs scoped to one task. Files with high collision risk: `project.yml`,
  `ShipController.swift`, `docs/TASKS.md` — rebase promptly if touched.
- CI builds every PR (`.github/workflows/build.yml`); a PR must be green before merge.
  Locally, build (or at minimum `swiftc -parse` the changed files) before opening a PR.
- Don't rewrite history on any pushed branch that isn't yours; never force-push `main`.
- Identify yourself in commits: Claude agents use their standard Co-Authored-By
  trailer; Codex should include `Co-Authored-By: Codex <noreply@openai.com>`.
- Secrets (future Claude/ElevenLabs keys) go in `Secrets.xcconfig` (gitignored) — never in code.
